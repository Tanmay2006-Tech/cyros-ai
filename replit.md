# Cyros AI

A full-stack AI-powered diet and fitness planner. Users enter their metrics and receive a personalized 7-day workout and diet plan generated by AI.

## Architecture

- **Frontend**: React + Vite, served via Express in development using Vite middleware
- **Backend**: Express.js (TypeScript, ESM)
- **Database**: PostgreSQL via Drizzle ORM
- **AI**: Supports OpenRouter API or Google Gemini API for plan generation; falls back to a randomized plan generator if no key is set
- **Monorepo layout**: `client/` (frontend), `server/` (backend), `shared/` (schema + API types)

## Project Structure

```
client/          React frontend (Vite)
server/
  index.ts       Express entry point (port 5000, 0.0.0.0)
  routes.ts      API route handlers
  engine.ts      AI plan generation (OpenRouter / Gemini / fallback)
  storage.ts     Database access layer
  db.ts          Drizzle + pg connection
  vite.ts        Vite dev middleware setup
shared/
  schema.ts      Drizzle schema (users, plans, meals)
  routes.ts      API route definitions + Zod schemas
```

## Running

- Dev: `npm run dev` — starts Express on port 5000, Vite as middleware
- Build: `npm run build`
- Start (prod): `npm run start`
- DB push: `npm run db:push`

## Environment Variables / Secrets

| Key | Required | Description |
|-----|----------|-------------|
| `DATABASE_URL` | Yes | PostgreSQL connection string (auto-set by Replit) |
| `OPENROUTER_API_KEY` | No | Enables AI plan generation via OpenRouter |
| `GEMINI_API_KEY` | No | Enables AI plan generation via Google Gemini |
| `SESSION_SECRET` | No | Session secret (optional if sessions added later) |

If neither AI key is set, the app generates randomized fitness plans as a fallback.

## Notes

- Originally deployed on Vercel; migrated to Replit
- `server/db.ts` handles SSL: enabled for Neon.tech connections, disabled for Replit internal Postgres
- Seed data (user + initial plan + meal) is automatically created on first startup if the database is empty
